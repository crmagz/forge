# Git Workflow Standards

Before creating a branch, changing commits, rebasing, or preparing a pull
request, read every reference below. Treat these rules as requirements unless
explicit repository-local instructions conflict with them.

## References

- `references/rules/git-branch-workflow.md` for operation safety, default-branch protection, branch cleanup, and landing changes.
- `references/rules/git-conventional-commits.md` for commit message format and allowed types.
- `references/rules/git-pull-request.md` for PR titles, required content, checks, and GitHub CLI workflow.

Do not perform destructive or write Git operations without the developer's
explicit approval when the active environment requires it.
